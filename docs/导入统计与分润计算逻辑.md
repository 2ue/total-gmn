## 当前最重要的系统事实

- 金额字段在交易表中始终保存为正数；收支方向由 `direction` 单独表达。
- 导入后真正参与业务的不是“原始行”，而是 `QualifiedTransaction`。
- `internal_transfer=true` 的记录会保留在明细中，但不进入利润与分润净额。
- 报表、图表、分润并非完全共用一套公式；差异已在分文档中明确标出。
- 历史分润批次在 HTTP 接口层禁止删除；增量分润标记过的交易不能删除，相关手动批次也不能再编辑。

## 维护原则

- 以后凡是改动导入、分类、统计、分润任一逻辑，建议同一个 PR 同步更新对应文档。
- 若代码与文档冲突，以代码与测试为准，再回补文档，不反过来。
